### 1. TCREI 프레임워크
AI와의 상호작용을 최적화하기 위한 구조적인 접근 방식입니다.

- **Task (임무):** 명확한 역할(Persona)과 출력 형식(Output Format) 지정
- **Context (배경):** 상황, 제약 조건, 타겟 오디언스 등 배경 정보 제공
- **References (참고 자료):** Few-shot prompting을 위한 예시 데이터 제공
- **Evaluate (평가):** 결과물 검토
- **Iterate (반복):** 프롬프트 수정을 통한 최적화

### 2. MS 프롬프트 엔지니어링 기법
Microsoft에서 제안하는 GPT 모델 활용 최적화 기법입니다.

#### 핵심 원리
- GPT 모델은 질문에 답하는 것이 아니라, **다음에 올 단어를 확률적으로 예측**하는 방식으로 작동합니다.
- 프롬프트 작성은 과학보다 **경험과 직관이 필요한 예술**에 가깝습니다.

#### 프롬프트의 5가지 구성 요소
1. **지시 (Instructions):** 모델이 수행해야 할 구체적인 작업
2. **주요 콘텐츠 (Primary Content):** 번역이나 요약 등 작업을 수행할 대상 텍스트
3. **예시 (Examples):** 원하는 입력과 출력의 예시 제공 (Few-shot learning)
4. **신호 (Cue):** 모델의 답변 방향을 잡아주는 유도 문구
5. **보조 콘텐츠 (Supporting Content):** 결과에 영향을 주는 배경 정보 (사용자 취향 등)

#### Best Practices
- **구체성 (Be Specific):** 해석의 여지를 줄이고 범위를 좁힙니다.
- **반복 (Double Down):** 중요한 지시는 앞뒤에 반복해서 배치합니다.
- **순서 (Order Matters):** 모델은 최신 정보(뒷부분)를 더 잘 기억하는 경향이 있습니다.
- **출구 전략 (Give an "out"):** 정보가 없을 때 "모름"이라고 답하게 하여 환각을 방지합니다.
